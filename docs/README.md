## Documentation Standards

This project follows these documentation standards:

1. **Clear and Concise**: Documentation should be easy to understand
2. **Up-to-date**: Keep documentation current with code changes
3. **Examples**: Provide practical examples where applicable
4. **Cross-references**: Link to related documentation
5. **Version-specific**: Document version-specific changes in CHANGELOG

## Contributing to Documentation

When contributing documentation changes:

1. Follow the contribution guidelines in CONTRIBUTING.md
2. Update relevant sections in the appropriate files
3. Add cross-references where helpful
4. Update the table of contents if needed
5. Test any code examples provided

## Language

Documentation is maintained in English to ensure broad accessibility. Translations are welcome as separate files or in the repository wiki.
